# Управление изменениями
 Ситуация: В середине проекта по внедрению корпоративной системы заказчик запросил добавление нового модуля отчетности, что увеличит бюджет на 15% и срок на 3 недели. Команда уже работает по утвержденному плану.

Задание: Опишите пошаговый алгоритм ваших действий в рамках процесса управления изменениями. Какие документы необходимо обновить? Как вы будете коммуницировать изменения с командой и стейкхолдерами?

## Диаграмма ганта
На диаграмме Ганта показан план разработки корпоративной системы: сначала идут инициация, аналитика, разработка и тестирование базового ядра (с 25 мая по 30 июня), затем — отдельный этап с модулем отчетности (с 1 по 24 июля). ## 1. Опишите пошаговый алгоритм ваших действий в рамках процесса управления изменениями.

На основании диаграммы Ганта (блок «Управление изменениями: Модуль отчетности CR-001» с 01.07.26 по 24.07.26) алгоритм следующий:

- Зарегистрировать запрос заказчика как CR-001 в журнале изменений.
- Провести анализ влияния и оценку рисков (задача на 01.07.26, ресурсы: PM + Старший Бизнес-аналитик).
- Провести заседание Управляющего комитета и подписать Дополнительное соглашение (задача на 02.07.26, ресурс: PM).
- Выполнить проектирование структуры данных и витрин отчетов (3 дня с 03.07.26, ресурс: Архитектор систем).
- Выполнить разработку ETL-скриптов (4 дня, Backend) и программирование макетов отчетов (4 дня, Frontend) — обе с 04.07.26.
- Провести комплексное интеграционное тестирование отчетности (4 дня, QA).
- Провести приемо-сдаточные испытания всей системы (1 день, 24.07.26, PM + QA).
- Закрыть CR-001 и зафиксировать веху.

## 2. Какие документы необходимо обновить?

Необходимо обновить следующие документы:
- Журнал изменений (Change Log) — добавить CR-001.
- Дополнительное соглашение к договору (бюджет +15%, срок +3 недели).
- Календарный план (диаграмма Ганта) — на диаграмме уже обновлён.
- План управления проектом (разделы: расписание, бюджет, риски).
- Техническое задание (добавить раздел «Модуль отчетности»).
- План тестирования (добавить тест-кейсы для отчетов).
- Реестр рисков (добавить риски интеграции и сдвига сроков).
- План коммуникаций (уведомления о новом этапе).

## 3. Как вы будете коммуницировать изменения с командой и стейкхолдерами?

**С командой:** объявить на ежедневном стендапе 01.07.26 о старте CR-001, разослать письменное уведомление в чате проекта / Jira с перечнем новых задач (ETL, макеты отчетов, тестирование), назначить ответственных согласно таблице (Backend, Frontend, QA, Архитектор), обновить доску задач.

**Со стейкхолдерами:** провести заседание Управляющего комитета 02.07.26 для подписания Дополнительного соглашения, направить официальное письмо заказчику об одобрении изменения и новом сроке сдачи (24.07.26), еженедельно направлять отчет о прогрессе по CR-001 всем стейкхолдерам.
